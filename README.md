Шаг 1 - Событие. Один из микросервисов (корзина, статус заказа, реклама) публикует событие.

Шаг 2 - Очередь. RabbitMQ или Kafka принимает событие и надёжно хранит, пока его не обработают.

Шаг 3 - Сервис уведомлений. Забирает событие из очереди, проверяет - подписан ли пользователь на такой тип уведомлений, рендерит текст по шаблону.

Шаг 4 - Отправщик. Отправляет готовое уведомление через FCM (Android) или APNs (iOS) на устройство пользователя.
